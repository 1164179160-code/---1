function GradientBg(props) {
  var colors = props.colors || ['#667eea', '#764ba2'];
  var layers = 8;
  var views = [];
  for (var i = 0; i < layers; i++) {
    var ratio = i / (layers - 1);
    var r1 = parseInt(colors[0].slice(1,3),16), g1 = parseInt(colors[0].slice(3,5),16), b1 = parseInt(colors[0].slice(5,7),16);
    var r2 = parseInt(colors[1].slice(1,3),16), g2 = parseInt(colors[1].slice(3,5),16), b2 = parseInt(colors[1].slice(5,7),16);
    var r = Math.round(r1 + (r2-r1)*ratio), g = Math.round(g1 + (g2-g1)*ratio), b = Math.round(b1 + (b2-b1)*ratio);
    views.push(
      <View key={i} style={{position:'absolute',left:0,right:0,top:Math.round((props.h||400)/layers*i),height:Math.ceil((props.h||400)/layers)+1,backgroundColor:'rgb('+r+','+g+','+b+')'}} />
    );
  }
  return <View style={[{position:'relative',overflow:'hidden'},props.style]}>{views}{props.children}</View>;
}

交互与动画

核心动效参数

页面转场：Animated.timing, duration 300ms, easing ease-in-out
卡片展开：Animated.spring, tension 80, friction 12（弹性自然，无过冲）
Tab 切换：Animated.timing, duration 200ms（快速响应，不拖沓）
所有动画使用 useNativeDriver: true，确保 60fps 流畅渲染

项目概述

极速骑行是整个协作的起点。野野提出了一个高端山地骑行 App 的概念，小野从零开始用 React Native 实现，经历了 7 个版本的持续打磨。

设计语言以深色沉浸式为核心，强调运动感和速度感。每个版本都在动画流畅度、视觉层次、交互细节上做了优化。

设计亮点

  • 深色沉浸式 UI，营造专业骑行氛围
  • 丝滑的页面转场和卡片动画，60fps 流畅体验
  • 多 Tab 导航：首页、车型、配件、发现
  • 渐变色系统与品牌色统一
  • 适配 Snack Expo 在线预览，纯 JS 无 TypeScript

版本演进

App.tsx → App-Simple.tsx → App-Fixed.tsx → App-Premium.tsx → App-Final.tsx → App-Exact.tsx → App-Perfect.tsx，每一版都是对上一版的精修和提升。从基础布局到动画打磨，从粗糙原型到像素级品质。

技术实现

基于 React Native + Expo，使用 Animated API 实现所有动画效果。为了兼容 Snack Expo 的限制，全部使用纯 JavaScript（无 TypeScript 类型注解），不依赖 expo-linear-gradient 等第三方库。

// 核心技术栈 React Native + Expo Animated API (spring / timing) 纯 JavaScript（Snack 兼容） 无第三方动画库依赖 7 个版本持续迭代

经验教训

这个项目最大的教训是"版本过多"。7 个版本意味着大量重复工作，如果一开始就分析透彻、规划清楚，可能 3 个版本就能达到最终品质。这也是后来建立"分析→规划→执行→自检→交付"工作流的直接原因。
